# 2cemestrforlabC++
Лабораторная работа по №1 команды "Анаконды"

=========Описание файлов проекта=========

classes - папка с реализацией сущностей Matrix, Point и Dataset. 
Функционал:
- Point: параметры x, y и label. возвращает [x, y] как вектор.
- Matrix: параметры lows и cols. получает элемент по номеру строки и столбца, устанавливает значение элемента по номеру строки и столбца, сложение, вычитание, умножение 2 матриц, умножение матрицы на число, возвращает количество строк, возвращает количество столбцов, транспонирование, заполняет всю матрицу значением value, заполняет случайными числами, применяет функцию к каждому элементу, выводит на экран, возвращает как строку.
- Dataset: добавляет точку типа Point, добавляет точку по координатам, получает точку по индексу, выводит количество точек, очищает датасет, все координаты в матрицу Nx2, все метки в матрицу Nx1

classes_tests - тесты для классов 
  
get_points.cpp и get_points.h - реализация функции, генерирующей датасет точек, разделенных на два равноудаленных от заданной (идеальной) прямой кластера. кластер с лейблом 1 находится выше прямой, с лейблом 0 - ниже.

vizualization.py - визуализация кластеров на координатной плоскости. берёт параметры точек из build/points.json. сохраняет png-файл (clusters.png) в папке build.

main.cpp - основная программа. генерирует датасет точек, кластеры расположены относительно прямой y = -2x + 10 (идеальная прямая). сохраняет датасет в виде json-файла в папке build (build/points.json). с клавиатуры вводятся коэфициенты k и b проверяемой прямой y = kx + b. предполагает лейблы точек по принципу: выше проверяемой прямой - 1, ниже - 0. считает total loss - процент ошибочного определения лейбла у точки - и выводит на экран. 


=============Запуск main.cpp=============

cd build

rm -f *

cmake ..

make 

./main


==========Запуск vizualization.py==========

cd .. (если начинаем из директории build)

python3 vizualization.py


===========================================

build/points.json и build/clusters.png нужны для наглядности и лежат в build, поскольку с каждым перезапуском main (после rm -f *) они удаляются и создаются заново с новыми значениями (clusters.png создаётся только после запуска vizualization.py)
